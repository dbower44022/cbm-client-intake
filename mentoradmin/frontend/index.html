How to set this up

A one-time setup by someone with access to both the organization's Google Cloud Console and the Google Workspace Admin console (a super-admin). Takes ~15 minutes.

In Google Cloud Console (console.cloud.google.com)

  1. Enable the Admin SDK API. APIs & Services → Library → search “Admin SDK API” → Enable.
  2. Create a service account. IAM & Admin → Service Accounts → Create. Give it a name (e.g. cbm-mentor-mailboxes). It needs no GCP roles — its access comes from the Workspace delegation below.
  3. Create a JSON key. Open the service account → Keys → Add key → Create new key → JSON. A file downloads — this is the key you paste into the form. Keep it secret.
  4. Copy the Client ID. On the service account's Details tab, copy the Unique ID (a long number) — you'll need it in step 5.

In Google Workspace Admin (admin.google.com, as a super-admin)

  1. Add domain-wide delegation. Security → Access and data control → API controls → Manage Domain-Wide Delegation → Add new. Paste the Client ID from step 4.
  2. Authorize these scopes (comma-separated), then Authorize:
    https://www.googleapis.com/auth/admin.directory.user.readonly
    https://www.googleapis.com/auth/admin.directory.user
    The first (read-only) is enough to check; the second (read-write) is required to create mailboxes. Add both.
  3. Pick the admin to impersonate. Any admin who can read the directory works for checking; to create mailboxes they need the User Management admin privilege. Use that email as Delegated Workspace admin email.

Back here

  1. Paste the JSON key and enter the delegated admin email.
  2. Tick Check… (and Create… if you want missing mailboxes auto-created), then Save.
  3. Click Test connection to confirm it works. Delegation can take 5–10 minutes to take effect — if the test fails right after setup, wait a few minutes and retry.
